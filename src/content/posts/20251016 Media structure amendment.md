#mediaStructure #systems  

We've created a categorized grouping system for the media structure. 

What this means is we will be creating packages arounds subjects that need  to be documented to make marketing content out of. The reason we chose this is to avoid encouraging long-form content where short-form is more appropriate.

### What is a category? 

In this instance we use the word category to refer to the subject were creating media about. The subject / the thing we are making media about defines the category it lives in. 

Categories are for material. 

Definition of each category:
#### Events:
Pretty self explanatory, whenever FBR throws an event we document.
	- CnC
	- X on Y

#### Activity:
This is about the activity / the things FBR is doing. This is also should be our primary category of media. 
This is to be treated as a communication channel to those who don't work internally with FBR. 
An example of a subject that fits into this category is FBR ROMs tech development/ product...
-  M dot 
- The Dial 
- Machine Hall 
FBR does cool shit so we want to document and release it as content since we don't do media marketing specifically, we don't need to do that because what we are working on is interesting enough. 
Activities should be the primary format of content and operate in bi-weekly cycles to maintain schedule.

#### Campaigns: 
This is for the bits. The media that isn't the core of FBR's activity but supports the activity.
An example is ([the nut milk penis video](https://www.instagram.com/reel/C6fOLwEPN7k/?utm_source=ig_web_copy_link&igsh=MzRlODBiNWFlZA==). It was useful to our social media as an extra bit of content to support the stuff that FBR does / is doing and marketing content.
The bits that make sense for the marketing but don't necessarily fit into a ridged package of media.   
Campaigns should be optimized for their specific format.
> Campaigns support the narrative of the activities

### Our Deliverables Matrix:
We decided to recycle the idea of grouping from the [202412 CNC Matrix](202412%20CNC%20Matrix.md) 

Media project rules follow the linear project rules refer to: [Linear point energy point system](obsidian://open?vault=FBR-DOCS&file=06%20Systems%2F04.1%20linear%2FLinear%20Energy%20Estimates%20(points)) for break down. 

Grouping is for output structure. 
#### GROUP A: (strictly) 
- 1 short form 
- 1 Photo 

#### GROUP B: (at least)
- 1 long form 
- 1 short form 
- 1 Photo 
- whatever additional content that is available to make if appropriate 

#### GROUP C: (any or)
- 1 or more long forms
- 1 or more short forms 
- 1 or more photos 
- whatever additional content that is available to make if appropriate

An example of the grouping system is CnC.
CnC (category: event) is mostly in group B,
- We require 1 long form per artist 
- 1 - 2 short forms 
- 4-6 carousel videos
- 1 or more photos 
There is a fluidity to what is available to produce in media output but there is also a minimum expectation. 


Now that we understand how it is physically structured. How do we get ideas on what to make media on. We use/ create/ document what we call "sparks".
### What are sparks? 
Sparks are pretty much what it sounds like refer to this document for a better breakdown. [Sparks breakdown](obsidian://open?vault=FBR-DOCS&file=06%20Systems%2F04.3%20media%20systems%2Fspark) ==(probably needs an update)==

#### How we document Sparks:
We have created a sparks folder in this obsidian work vault under ==*02 interpreted design.*==
There is a ==*sparks template*== in the assets folder at the bottom of the notes list in the FBR-WRK vault .
Please use this template when documenting a spark so we can transform it into media and link everything to reference easily when completing projects. 

In this matrix, sparks are assigned a groups in linear. This way we can make these into achievable projects and tasks to get done. 

### How do we use Obsidian for media: 
We use Obsidian for 2 things. 
- Sparks documentation
- Production documentation 
The reason we use obsidian is so we can make notes on the media we are making and have a sort of instruction manual for how to build our marketing material. 
This is where the details live. 
We document everything here so that anyone on the FBR ROM team can refer to it and understand what is going on. 

### How do we use Linear for media:
We use linear as our dictator. It tells us when we have to do something and when and the whole timeline for our projects but it doesn't really tell us how, which is why we use obsidian. Linear is only used to manage us. 
So we have templates for our media system already available in linear when creating a new project. 
These templates use the Group system that we defined earlier. 

### How do we use them together for media 
We always link everything within everything. 
There should always be a link to the Linear project in the obsidian files we use to determine the content we are making. 
And vice versa 
There should always be a Obsidian link in the Linear project to link back to how we are making the content that has been assigned to us in Linear.
